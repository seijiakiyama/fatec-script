Quinta Entrega

Os vídeos indicados para aula descreveram as formas de login na aplicação feita utilizando o Tekton.

São 3 opções, já programadas, de login:

Utilizando a conta Google, que já possibilita o acesso à página de admin, se utilizado o email que foi cadastrado no AppEngine;

Utilizando Passwordless, ou seja, apenas com o email já cadastrado na aba de administração e utilizando o Token disponibilizado na aplicação;

Utilizando o Facebook, onde é necessário criar um app no Facebook Developers e utilizar o Token gerado.



O framework, por padrão, é protegido.

Para desproteger a página, para acesso sem login, é utilizada a notação 

@login_not_required
.

Para desabilitar a proteção contra Cross-site request forgery Tipo de exploração maliciosa pelo qual comandos não autorizados são transmitidos de um usuário que confia no website., é utilizada a notação 

@no_csrf

Para aplicar esse conceito, uma página de formulário pode ser utilizada. Caso um método esteja com a notação acima, será desprotegido e acessível. Em casos de manipulação de dados, o método que os salva não deve ter a notação.

Através do Tekton, a página /permission já exibe as configurações de permissões.

Possuindo grupos de acesso, as permissões podem ser configuradas utilizando as notações 

@permissions()
@login_required
.

Por fim, pode-se utilizar uma variável já instanciada e utilizada na base, _logged_user. Através dele, pode-se obter informações do usuário logado, como é feito ao exibir a aba de ADMIN e o nome/email do usuário ao lado do botão de Logout.
