Segunda Entrega

Os vídeos propostos consistiram na configuração do ambiente a ser utilizado em sala de aula.

Para um suporte inicial, são utilizadas as ferramentas "pip" e "virtualenv". Ambas iniciam a estrutura que o framework "Tekton" usa.

Assim, o App Engine, da Google, e PyCharm, da JetBrains, tornam-se fortes instrumentos de desenvolvimento Web, contendo todas as bibliotecas e ferramentas de execução e debug.


1. Quais as duas principais versões atuais do interpretador Python?

As principais versões são 2.7.x (utilizada em aula) e 3.4.


2. Qual versão de Python é suportada atualmente pelo App Engine?

Versão 2.7.


3. Para que serve o SDK do App Engine?

A finalidade do SDK é fornecer acesso às ferramentas de desenvolvimento.


4. Qual a vantagem de colocar ferramentas instaladas no path do Sistema Operacional?

As ferramentas são relacionadas à variável PATH para seus comandos serem executados no CMD em qualquer diretório.


5. O que é e para que serve uma IDE?

Uma IDE fornece ferramentas de desenvolvimento que facilitam a programação e execução, indepentende do SO.


6. Como se chama o arquivo de configuração de uma aplicação App Engine?

app.yaml.


7. Qual o domínio padrão para acessar aplicações App Engine na internet?

appspot.com.




1. Qual o nome do arquivo de configuração do Google App Engine?

app.yaml.


2. Para que serve o item application do arquivo de configuração?

Identificar a aplicação, ou seja, deve ter o mesmo ID definido na criação dela.


3. Para que serve o item version do arquivo de configuração?

Identifica a versão da aplicação que deve ser utilizada na publicação.


4. Qual endereço deve ser utilizado para acessar uma aplicação com id foo e versão 35?

A suposta applicação teria o endereço http://35-dot-foo.appspot.com.br.


5. Para que serve a seção libraries do arquivo de configuração?

Configurar as bibliotecas utilizadas pela aplicação.


6. Para que serve a seção handlers do arquivo de configuração?

Mapear os scripts Python executados de acordo com o path das requisições.


7. Como são definidos os paths mapeados no arquivo de configuração?

São definidos por expressões regulares.


8. Por que é necessário mapear RequestHandlers nos scripts Python?

Em um arquivo, podem-se definir diversas classes. Então, o mapeamento via arquivo não é suficiente para definir a classe utilizada no processo de uma requisição.


9. Para que serve a classe RequestHandler?

Definir o código que processa uma requisição HTTP.


10. Como se relacionam os métodos da classe RequestHandler e os do protocolo HTTP?

O protocolo HTTP realiza chamadas, que ao serem processadas, estão relacionadas aos métodos de mesmo nome (Chamada GET, método get).


11. Para que serve o objeto Request?

Serve de interface de acesso à informações de uma requisição HTTP.


12. Como se obtém os valores de parâmetros enviados via query string em uma chamada HTTP do tipo GET?

Utilizando o método get do objeto Request.


13. Para que serve o objeto Response?

É uma interface de construção de uma resposta HTTP.


14. Qual o método do objeto Response serve para enviar strings?

Método write.


15. Como é possível enviar uma resposta para redirecionamento?

Utilizando o método redirect, do objeto RequestHandler.




1. Para que serve o Virtualenv?

Criar ambientes virtuais Python isolados.


2. Qual a função do arquivo convention.py?

Realizar a ligação entre o Tekton e o Webapp2.


3. Por que é necessário incluir bibliotecas através de código no arquivo convention.py?

O App Engine não tem acesso ao SO para instalar as bibliotecas usando o virtualenv e pip. Então, precisam ser incluídas no arquivo.


4. Como ficaria a declaração de uma função para tratar a execução de chamada no path /usuario/salvar?nome=Renzo&idade=31?

"def salvar(nome,idade)".


5. Como se diferenciam parâmetros recebidos por injeção de dependência dos recebidos via requisição HTTP?

Os parâmetros recebidos via injeção possuem o prefixo "_".


6. Qual deve ser a posição de parâmetros recebidos via injeção de dependência?

Os parâmetros recebidos via injeção de dependência devem ser os primeiros a serem declarados.


7. Qual deve ser o parâmetro declarado quando for necessário fazer um redirecionamento?

"_handler", para acessar a instância de um RequestHandler.


8. Qual o script e função devem ser utilizados para se calcular paths com base em uma função?

A função "to_path", do módulo "tekton.router".
